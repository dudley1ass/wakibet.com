Render Web Service — WakiBet API (monorepo root)
================================================

This repo is Node + pnpm at the REPO ROOT (no `backend/` folder).

Dashboard checklist
-------------------
[ ] Root Directory: EMPTY (not "backend", not "apps/api", not "frontend")
[ ] Environment: Node 20+ (not Python 3)
[ ] Build Command:  pnpm run render:build
[ ] Start Command:  pnpm run render:start
[ ] Env var DATABASE_URL: from Render Postgres (Internal URL if DB is on Render)
[ ] Pre-Deploy Command (optional but recommended):
      pnpm --filter @wakibet/api exec prisma migrate deploy

Render sets PORT automatically; the API listens on 0.0.0.0:$PORT.

Env var WAKIBET_JWT_SECRET: long random string (required for auth in production).

---

Static Site (browser UI — apps/web)
------------------------------------
[ ] Root Directory: EMPTY — delete ALL text in the box (not "frontend", not "backend",
    not "apps/web"). If Render still says 'frontend' does not exist, the box is not empty.

[ ] Build Command (pick one):
      pnpm run render:build:web     (fastest — only installs + builds the web app)
    or  pnpm run build              (builds shared + api + web; skips Expo mobile)
    Wrong: NOT bare "turbo build" alone; NOT render-build:web (wrong punctuation).

[ ] Publish directory (copy exactly):
      apps/web/dist

Build-time env (required or the site will call /api on the static host and login will fail):
[ ] VITE_API_BASE = https://YOUR-API.onrender.com   (no trailing slash)

Client-side routes (required or /contact, /scoring-table, refresh on any path → 404):
[ ] Static Site → Redirects / Rewrites → add a Rewrite (not Redirect):
      Source: /*
      Destination: /index.html
    Render docs: https://render.com/docs/redirects-rewrites

Still failing? Open the failed deploy → Logs and search for ERR / error / command not found.
Common: pnpm missing (corepack is enabled by the script above), or wrong publish path.

After saving, trigger Manual Deploy.
